# Contributing to this project

Thank you for your interest in contributing. This project is volunteer-built open source software for the fencing community. Contributions of all kinds are welcome — bug reports, hardware testing feedback, code improvements, and documentation.

---

## Before you contribute

All contributors must sign a Contributor Licence Agreement (CLA) before their contribution can be merged. This is a one-time step per contributor, covering all future contributions to this repository.

See the [Signing the CLA](#signing-the-cla) section below for instructions.

---

## How to contribute

### Reporting bugs or issues
Open a GitHub issue. Please include:
- What you were doing when the problem occurred
- What you expected to happen
- What actually happened
- Hardware details if relevant (board variant, wiring, power supply)

### Suggesting improvements
Open a GitHub issue before starting work on a significant change. This avoids duplicated effort and ensures the change fits the project direction.

### Submitting code
1. Fork the repository
2. Create a branch for your change (`git checkout -b my-change`)
3. Make your changes
4. Test on real hardware where possible
5. Open a pull request with a clear description of what you changed and why

### Armourer and field feedback
If you have used this device in a real armoury or competition context, feedback on usability, reliability, and test accuracy is especially valuable — even if you don't write code. Open an issue or reach out via LinkedIn.

> ⚠️ Pull requests from contributors who have not signed the CLA will not be merged.

## Code style

- Language: C++
- Build system: PlatformIO
- Follow the conventions already present in the codebase
- Keep changes focused — one logical change per pull request
- Comment non-obvious logic, especially anything related to timing, resistance thresholds, or hardware behaviour
